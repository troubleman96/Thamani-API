# alembic — Database Migrations

Manages PostgreSQL schema migrations using Alembic with async SQLAlchemy support.

---

## Configuration

`alembic.ini` sets `sqlalchemy.url` as a fallback. In practice, `env.py` reads `settings.DATABASE_URL` at runtime so the actual URL always comes from `.env`.

---

## `env.py`

The key difference from a standard Alembic setup is async engine support:

```python
async def run_async_migrations() -> None:
    engine = create_async_engine(settings.DATABASE_URL)
    async with engine.begin() as conn:
        await conn.run_sync(do_run_migrations)
    await engine.dispose()

def run_migrations_online() -> None:
    asyncio.run(run_async_migrations())
```

All SQLModel table models are imported at the top of `env.py`. This is required for `--autogenerate` to detect schema changes — if you add a new model and forget to import it here, Alembic will not generate the migration for it.

## Common commands

```bash
# Generate a migration after changing models
alembic revision --autogenerate -m "add startup_views column"

# Apply all pending migrations
alembic upgrade head

# Roll back one migration
alembic downgrade -1

# Show current migration state
alembic current

# Show migration history
alembic history --verbose
```

---

## Adding a new model

1. Create the model in `app/apps/<yourapp>/models.py` as a `SQLModel` with `table=True`
2. Add an import to `alembic/env.py`:
   ```python
   from app.apps.yourapp.models import YourModel  # noqa: F401
   ```
3. Run `alembic revision --autogenerate -m "add your_model table"`
4. Review the generated file in `alembic/versions/`
5. Run `alembic upgrade head`

---

## `versions/`

Auto-generated migration files live here. Each file has an `upgrade()` and `downgrade()` function. Review every autogenerated migration before applying — Alembic can miss certain changes (column type changes, enum additions) that require manual edits.
